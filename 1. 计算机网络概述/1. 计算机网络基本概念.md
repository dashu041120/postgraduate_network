## 1. 概念
计算机网络中，常用的节点为第一个节点
节点一般指有处理能力的实体
结点一般为一个标记，数据结构中

## 2. 组成

1. 从组成上看，有以下三个部分：
	1. 硬件
	2. 软件
	3. 协议
2. 从工作方式上看
	1. 边缘部分：供用户直接使用的主机组成，（端系统），**端到端**的通信
		1. 客户/服务器方式 C/S
		2. 对等方式P2P
	2. 核心部分：由路由器组成，他是核心关键构件：==提供服务==

## 3. 功能
1. 数据通信
2. 资源共享
3. 提高可靠性
4. 分布式处理
5. 负载均衡：提现了分布式处理的思想

## 4. 分类

- 按覆盖范围分类
	- 广域网
	- 城域网
	- 局域网
	- 个人区域网
- 按拓补结构分类
	- 星形：单点故障不会影响全网，中心节点负载高
	- 环形：增减节点容易，单点故障会引起全网故障
	- 总线形：电缆数量少，易于扩充，网络效率和贷款利用率低
	- 网状：有很多冗余链路，可靠性高，延迟低，线路布置成本较高，结构复杂，不易维护
- 按交换技术分类
	- 电路交换
		- 建立链接－传输数据－释放连接
	- 报文交换
		- 原地址+目的地址
			- 存储转发网络
		- ==最小信息单位是报文！==
	- 分组交换
		- 将较长的报文划分成多个更小且尽可能等长的数据段
		- 采用存储转发技术
		- ==最小信息单位是分组！==
## 5. 性能指标
- 速率
- 带宽  hz
- 吞吐量，实际传输的比特数
- 时延
	- 处理时延
		- 路由器的计算能力
		- 通信协议的复杂程度
	- 排队时延
		- 等待队列的分组数量
		- 端口的发送速率
	- 传输时延 = 分组长度 / 发送速率
	- 传播时延 = 信道长度 / 传播速率
- 时延带宽积 = 传播时延 x 信道带宽
- 往返时延 RTT
- 利用率
	- 信道利用率：不是越多越好
	- 网络利用率
